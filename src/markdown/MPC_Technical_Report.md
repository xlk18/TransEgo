# 四旋翼非线性模型预测控制器 (Quadrotor NMPC) 技术报告

## 1. 模块概述 (Overview)

本项目提供了一套面向四旋翼无人机的非线性模型预测控制器（NMPC）。整个系统基于 `CasADi` 符号求导与优化框架，底层调用 `Ipopt` 求解器进行实时非线性规划（NLP）求解。在复杂的敏捷飞行与轨迹追踪任务中，该控制器提供了基于系统物理动力学的高精度前馈预测预测与最优反馈控制。
为适应不同的算力平台与任务需求，该包内置了**基础版（Normal MPC）**与**高级版（Advanced MPC）**两套隔离部署的节点代码。
---

## 2. 控制管线体系 (Pipeline)

无论是基础版还是高级版，控制器都遵循以下闭环数据流（Pipeline）运行：
1. **状态更新 (State Feedback)**
   - 通过订阅 `nav_msgs::Odometry` 实时获取四旋翼当前的全局位置、线速度以及姿态（四元数）。
   - 【高级版专属】在状态机中缓存上一时刻下发的指令（推力与角速度），用作系统一阶延迟模型的输入初始状态。
2. **参考轨迹解析 (Trajectory Parsing)**
   - 订阅上层规划器生成的 `trajectory_msgs::MultiDOFJointTrajectory`。
   - 解析未来 $N$ 个步长内各个时间节点上的期望位置、期望速度、期望加速度以及期望姿态（偏航角与微分平坦重构姿态）。
   - 利用牛顿第二定律将参考加速度转换为基础前馈推力 $T_{ref} = m \|a_{ref} + g\|$，形成控制量参考数组。
3. **非线性规划求解 (NLP Solving)**
   - **打靶法 (Multiple Shooting)**: 将预测时域 $N$ 内的控制与状态全部设为决策变量。
   - **积分器**: 基于系统动力学微分方程 $\dot{x} = f(x, u)$，采用**四阶龙格-库塔 (Runge-Kutta 4) 法**执行多步高精度数值积分，确保预测模型的物理准确性。
   - **代价函数**: $J = \sum (x_k - x_{ref})^T Q (x_k - x_{ref}) + (u_k - u_{ref})^T R (u_k - u_{ref}) + J_{terminal}$
   - **热启动 (Warm Start)**: 将上一步求得的控制序列与状态序列后移一拍，作为本次求解迭代的初值，大幅度降低 Ipopt 收敛耗时。
4. **控制指令投递 (Command Dispatching & Mapping)**
   - 提取出求解出来的第 $0$ 步最优控制量（推力 $T$ 与三轴机体系角速度 $\omega_x, \omega_y, \omega_z$）。
   - 将具备物理量纲的推力 $T$（单位 Newton），基于设定的物理边界 `[min_thrust, max_thrust]` 进行 **标幺化映射 (0.0 ~ 1.0)**，组装至 `mavros_msgs::AttitudeTarget` 格式下发至飞控（如 PX4）。
   - 将预测到的未来 $N$ 步状态封装为 `nav_msgs::Path` 并发布至 RViz 供直观调试使用。

## 3. 外部接口协议 (Interfaces)

### 订阅话题 (Subscribers)

| 话题名称 | 消息类型 | 功能说明 |

| --- | --- | --- |

| `/Odometry` | `nav_msgs::Odometry` | 接收 VIO 或 MOCAP 系统提供的融合里程计状态估测 |

| `/trajectory` | `trajectory_msgs::MultiDOFJointTrajectory` | 接收运动规划系统（如 Ego-Planner）下发的安全跟踪轨迹 |

### 发布话题 (Publishers)

| 话题名称 | 消息类型 | 功能说明 |

| --- | --- | --- |

| `/mavros/setpoint_raw/attitude` | `mavros_msgs::AttitudeTarget` | 将推力比例和角速度旁路发布给基于 MAVROS 的底层飞控 |

| `/nmpc_predicted_path` | `nav_msgs::Path` | 面向 RViz 的 NMPC 预测窗口 $N$ 内的最优轨迹渲染 |

## 4. 关键物理与调优参数 (Parameters)

| 参数名 | 默认值 | 意义 |

| --- | --- | --- |

| `mass` | 1.0 | 无人机重量 (kg) |

| `gravity` | 9.81 | 重力加速度 ($m/s^2$) |

| `dt` | 0.05 | MPC 离散步长 (s) |

| `N` | 20 | 预测视界长 (Horizon)；即向前预测 $20 \times 0.05 = 1.0s$ 轨迹 |

| `max_thrust` / `min_thrust` | 20.0 / 1.0 | 底层电调可输出的最大/最小推力 (N)，用于控制量截断与向 PX4 的 $[0, 1]$ 归一化映射 |

| `max_bodyrate_[x,y,z]` | 3.0 / 3.0 / 1.5 | 飞控支持的最大机体系横滚、俯仰、偏航角速度限制 (rad/s) |

| `Q_diag` / `R_diag` | 视版本而定 | 运行代价的对角权重阵，$Q$ 衡量状态跟踪精度，$R$ 惩罚激进控制导致的能耗抖动 |

| `terminal_cost_multiplier` | 10.0 | 终端阶段代价值的放大系数（用于 Advanced 版本的无限时域 LQR 惩罚稳定机制） |

## 5. 版本详细对比：基础版 vs 高级版

针对不同阶段对于四旋翼控制器的需求，库中保留了两套 NMPC 实现方案（`quadrotor_nmpc` 与 `quadrotor_nmpc_advanced`）。两者的接口一致，但动力学底座和优化理论有巨大差异。
### 📌 基础版 NMPC (Normal MPC)

- **定位**：轻量级、简易调试、主要应付悬停与一般慢速/匀速航线跟踪任务。
- **状态空间**：多采用标准质点系 / 简易欧拉角。
- **动力学模型**：标准刚体运动学，忽略空气动力学阻力以及底层执行机构的延迟。
- **优化策略**：使用经典固定权重的二次型代价函数求解，若无特殊构造往往缺乏强有力的末端状态稳定性保障（Terminal Control Invariant Set），但在短视界、低速段够用。
- **优点**：求导矩阵极小，运算飞快；对于算力受限（如极小型无机载电脑的伴随式无人机）具有较好妥协。
### 🚀 高级版 NMPC (Advanced MPC)

- **定位**：为**高敏捷飞行**与**剧烈机动**量身定制的工作于严苛物理边界下的追踪控制器。
- **扩展的状态空间 (14维)**：
  - 除了追踪 `p(3)` 和 `v(3)` 外，姿态上引入四元数 `q(4)` 以**回避大机动下的万向节死锁**。
  - 在状态中增加了 `T(1)`(推力状态) 和 $\omega(3)$(机体系角速度状态)，构成共 $14$ 维空间：$x = [p, v, q, T_{act}, \omega_{act}]^T$。
- **拟真物理动力学**：
  - **气动阻力补偿** (`drag_x, drag_y, drag_z`)：当无人机以大于 $5m/s$ 速度飞行时，转子气动阻力激增。高级模型将其完整融入微分平坦运动学：$dv = R \cdot [0, 0, T]^T - g - D \cdot v$。
  - **一阶惯性延迟系统** (`tau_thrust, tau_rate`)：利用延迟常数建立方程 $\dot{\omega} = (\omega_{cmd} - \omega)/\tau$，告知优化器执行机构（电机）无法瞬间达到指令值，使 NMPC 提前做出超前“预留位”控制。
- **无限时域的终端 LQR 惩罚接驳 (LQR Terminal Cost)**:
  - 为了彻底保障闭环系统的收敛稳定性，Advanced 版在构建 NLP 之前利用特征点（悬停平衡点）提前建立 $9$ 维切空间误差的连续系统雅可比阵 $A_c, B_c$。
  - 使用 DARE (离散代数黎卡提方程) 收敛迭代求得系统的终端理论稳定代价矩阵 $P$。
  - 将 $x_N^T P x_N$ 注入最终代价函数，从而将有限时域 NMPC 问题平滑延伸至无限时域 LQR 控制下界。
- **SO(3) 流形状态偏差**:
  - 计算代价时，不直接相减四元数（该计算方式在非凸域里物理意义弱），而是构造误差四元数 $q_{err} = q_{ref}^{-1} \otimes q$，从中提取微小旋转的李代数差向角投影作为真正的 $3$ 维姿态惩罚依据，显著增幅了姿态大偏离状态下的修正能力。
